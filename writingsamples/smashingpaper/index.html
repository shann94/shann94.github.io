ATOM V. DREAMWEAVER: DAWN OF HTML CODING

If there is one thing I have learned in my college career maneuvering through the Communication and English Department at Virginia Tech, there are several ways to do one thing. This can be applied to one program (e.g., there’s many different ways to make text bold in Adobe InDesign) or one concept (e.g., coding for a website).

When it comes to web design, one can use What You See Is What You Get (WYSIWYG) editors like Adobe Dreamweaver or code mode editors like Atom. Today, you're going on a journey to uncover which editor is the right one for you based on the facts, rather than my opinion (although I will give my opinion at the end).

WYSIWYG

Overview: WYSIWYG editors are programs that have the ability to directly manipulate the layout of a document without typing or remember names of layout commands. Web designer, Julia Blake, wrote in "Building a Website with WYSIWYG Editors: Pros and Cons," "This type of editing tool is fast, easy to manage, and immediately gratifying: you immediately see the result of the changes you apply to the website design."

Essentially, there is no actual coding here; therefore, users do not understand the language of coding, but is that necessary if the user is not planning on becoming a web designer? Web designer, Jennifer Kyrnin, wrote in her article "Pros and Cons of Text Editors," "The problem was (and often, still is) that WYSIWYG editors can generate HTML that is hard to read, not standards compliant and only really editable in that editor."

However, for someone who is strictly into design, it is argued that WYSIWYG programs enables users to design with ease because their main goal is a "visuals first" design. It is not important to learn code because the design element is what is most important. A benefit of WYSIWYG editors is that elements can be manipulated and arranged with ease. This makes production of mobile pages easier to do. The production of a webpage is faster and instantly gratifying because the designer does not have to debate whether their code will work or not.

Web designer, Joshua Johnson, argues against this in his article "Why Adobe Doesn’t Understand Web Designers", "The web... is built on interaction. The basic idea of the worldwide web as we now know it is that it’s an interactive portal to a global network. No matter how much current and former print designers, myself included, want it to be print design on a screen, that’s simply not the case. The web is a living breathing thing and print design metaphors taken too far are simply serving up dead, static content."

In WYSIWYG programs, like Adobe Dreamweaver, the design of a webpage generates a HTML code for a webpage. The manipulation on the preview page changes the code on the coding page to ensure that the manipulation is saved. However, the code can be messier than if the webpage was coded by hand.

Pros of WYSIWYG

  • Easier to use than code mode editors,
  • Enables "visual designers" to design content without learning new skills. Designers can focus on design rather than asking "does my code work?"
  • Fast; less time and effort spent,
  • Users have more control,
  • Immediately gratifying,
  • WYSIWYG editors do generate code for future editing by a code designer.

Cons of WYSIWYG

  • Typically cost money, especially for those programs that are actually good,
  • WYSIWYG editors can generate inaccurate code or the code is messy making ti harder to edit in the future,
  • It is hard to move from one text editor to another (e.g., cannot move from Dreamweaver to Froala with ease),
  • Web is an interactive place.

Code Mode

Overview: Code mode editors are editing programs that enable users to write code using all the brackets and forward slashes the heart desires. If you plan on coding for a long time, this is probably the best place to start.

Understanding how HTML code actually works enables you to see an error and be equipped with the skills to fix it quickly, rather than scanning 10,000 lines of code and not having a clue what you're looking for. Your coding skills will become more sharp over time when new experience arise. Shay Howe wrote in her HTML and CSS lesson titled, "Performance and Organization," "As a website’s code base and traffic grows, a new skill set comes into play, one that is extremely important to both development time and user experience."

Basically, if you want to do HTML coding for a living, you should probably know how code mode editors work and the HTML language. Yes, it's called a language, but it isn't as scary as it sounds. Expert web designer, Christopher Heng, wrote in his article titled, "Should I Learn HTML or Just Use a WYSIWYG Web Editor? Pros and Cons of Using a Visual Web Editor vs Learning HTML" that, "HTML is actually a very easy document formatting language to learn. It was deliberately designed that way. Even if you're an incidental webmaster, and not a computer geek, the language can usually be mastered by many people with little effort." Heng would be correct; the language is still in English, there is just a mix of symbols involved.

Howe adds that it's time consuming to render a web page from scratch. However, she notes, "Taking the time to design the right structure for a code base, and identify how all of the different components will work together, can speed up production and make for a better experience all around." So although it takes time, this process is better for the coder and the users. Additionally, with experience the process speeds up and the website will perform better.

Kyrnin notes that some users of text editors claim that it is harder to design with code mode editors because they cannot see what the code actually looks like, compared to WYSIWYG editors. However, text editor applications, such as Atom, actually have a preview mode that enables the user to see the webpage as they are coding.

Additionally, some editors have advanced as far as color coding different tags for what they represent. For example, in Atom the tag 'class' will appear orange when the file is saved as html. Another nice feature of code mode editors is files can be Markdown, HTML, txt, etc.

Pros of using code mode editors

  • Being knowledgable in this skill boosts website performance
  • Faster edits: fixing errors is a smoother process
  • Typically free
  • Many different applications to choose from
  • Fluent in a new language
  • Can save files in many different formats

Cons

  • Must know the HTML language to effectively use code mode editors,
  • Takes time to initially code a web page,
  • More difficult to "design" if using an application lacking a preview mode
  • More complicated until user gets the hang of it

Who's the Real Winner?


That's right. Not only is it the most frustrating, but it is the most rewarding. If you're looking to be successful web designer, you gotta know what your code means and what you're doing.

My suggestion is this: if your interested in designing for a long time, learn code more. If you are designing a webpage for a very short period of time, use WYSIWYG editors. If you're unsure, starting learning code mode, but use WYSIWYG as a crutch. A mixture of the two will help with initial designs, but be sure you are not cheating yourself out of a valuable skill by leaning on WYSIWYG editors.
